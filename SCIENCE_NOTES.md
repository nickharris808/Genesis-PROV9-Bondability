# SCIENCE_NOTES.md -- PROV_9_BONDABILITY

## Audit Summary

A red-team science audit scored this module **4.0/10** and identified four
serious issues. This document records the root causes, the fixes applied,
and the remaining limitations that users must understand before relying on
yield predictions.

---

## Issue 1: ARCHIVE RUN PRODUCES yield=0.0 (CRITICAL)

### Root Cause

The archived demo layout (`archive/demo_layout.json` and
`archive/out_physics_final/`) was generated by an old version of the
synthetic layout generator that created **760 tiny random pads on a
2000x2000 um die**, producing **0.6% pattern density**. At this density:

1. CMP recess is ~15 nm (at the low-density extreme of the calibration
   curve), producing a gap of ~15.5 nm.
2. The bonding snap-in threshold is ~9 nm (Dugdale cohesive zone model).
3. Every tile has gap >> snap-in, so void risk is ~100% everywhere.
4. With all clusters at ~100% failure, the product model gives yield = 0.0
   exactly, for all Monte Carlo trials.

This is a parameter problem, not a code bug. The model correctly predicts
that a layout with 0.6% density cannot bond -- but 0.6% density does not
represent any real hybrid bonding design.

### Fix Applied

1. **Synthetic layout generator** (`bondability/demo/synthetic.py`): Already
   fixed prior to this audit. Now generates a 1600x1600 um die with a
   4 um-pitch regular pad array (~45% density), routing channels, and edge
   keep-out zones. This is representative of real Cu-Cu hybrid bonding
   layouts (TSMC SoIC, Intel Foveros).

2. **Buyer verification script** (`bondability/verification/run_buyer_verification.py`):
   Updated the fallback process YAML to use `copper_hybrid_bonding` CMP
   calibration (recess ~2.5 nm at optimal density) instead of standard Cu
   damascene values (recess ~8 nm). Added a `yield_nonzero` check that
   catches yield=0.0 on realistic layouts.

3. **Pipeline diagnostic** (`bondability/pipeline.py`): Added a yield=0.0
   warning that reports density_mean, gap_mean, and void_risk_mean to help
   users diagnose the cause.

4. **Archived yield_summary** (`archive/out_physics_final/yield_summary.json`):
   Annotated as stale data from the old broken layout. The archive data was
   NOT regenerated because it would require running the full pipeline; the
   annotation explains that the current code produces yield_p50 in the
   0.70-0.95 range with the fixed layout.

5. **Smoke test** (`tests/test_smoke.py`): Added assertion that yield_p50 >
   0.01 on the demo layout to catch future regressions.

### How to Verify

Run the smoke test:
```
pytest tests/test_smoke.py -v
```

Or run the pipeline manually:
```python
from bondability.demo.synthetic import generate_synthetic_layout
from bondability.pipeline import run_pipeline
from pathlib import Path

layout = generate_synthetic_layout(seed=0)
result = run_pipeline(
    gds_path=None,
    process_yaml=Path("configs/process_example.yaml"),
    out_dir=Path("/tmp/bondability_test"),
    layout_override=layout,
)
print(result["yield_summary"])
# Expected: yield_p50 between 0.70 and 0.95
```

## Issue 2: correlation_length DOMINATES YIELD BY ~100x (CRITICAL)

### Root Cause

The `correlation_length_um` parameter controls how many independent failure
clusters exist on the die:

```
n_clusters = (H * W) / (correlation_length / tile_um)^2
```

With fewer clusters (higher correlation_length), failures are correlated and
yield is higher. With more clusters, each acts as an independent failure
source, rapidly reducing yield. Changing `correlation_length_um` from 100
to 2000 can swing yield from ~40% to ~99%, while all other sampled
parameters (overlay, particles, roughness) combined affect yield by only a
few percent.

**This means the entire yield prediction is essentially determined by one
parameter that is uncalibrated by default.**

### Fix Applied

1. **Extensive documentation in config** (`bondability/config.py`, lines
   135-175): Added a block comment explaining what the parameter is, what
   experimental measurement corresponds to it, how to calibrate it (4
   methods in decreasing reliability), and the realistic range (100-2000
   um).

2. **Sensitivity sweep** (`bondability/yield_model/model.py`,
   `_correlation_length_sensitivity()`): Every pipeline run with
   `sensitivity_analysis=True` now sweeps correlation_length across [100,
   200, 500, 1000, 2000] um and reports the yield swing. The output
   includes a `correlation_length_dominance_warning` string.

3. **Data-driven estimate** (`bondability/yield_model/model.py`,
   `estimate_correlation_length()`): Computes the spatial autocorrelation
   of the density map and finds the 1/e decay distance. This is a layout-
   based proxy for defect correlation length. The yield summary now reports
   both the configured and data-driven estimates, plus a calibration note.

4. **Error budget** (`bondability/error_budget.py`): The yield_distribution
   solver is marked as `calibration_status="uncalibrated"` and reports the
   mismatch between configured and data-driven correlation lengths.

### What Experimental Measurement Corresponds to correlation_length_um

`correlation_length_um` is the spatial autocorrelation length of bonding
defects (voids, non-contacts) on the bonded wafer surface. It can be
measured by:

- **Scanning Acoustic Microscopy (SAM)** of bonded wafer pairs
- **IR transmission imaging** of bonded interfaces
- **Cross-section SEM** at multiple die locations

Procedure: Create a binary defect map from inspection data. Compute the 2D
autocorrelation function. The distance at which the autocorrelation drops to
1/e (~0.368) is the correlation_length.

### Realistic Range

| Source | Range (um) | Notes |
|--------|-----------|-------|
| Cunningham 1990 | 200-500 | Generic IC defects |
| Turner & Spearing 2002 | 100-300 | Direct bonding defects |
| Typical fab data | 200-800 | Cu-Cu hybrid bonding |
| Default value | 500 | A guess -- must be calibrated |

### Remaining Limitation

**Yield predictions are ONLY as reliable as the correlation_length
calibration.** Until this parameter is calibrated to fab defect data, yield
values should be treated as relative rankings (layout A vs layout B) rather
than absolute predictions.

## Issue 3: 7-SOLVER CHAIN FRAGILITY

### Root Cause

The pipeline chains 7 solvers in sequence:

1. Feature Extraction (layout -> density map)
2. CMP Recess Prediction (density -> recess/gap)
3. Contact Mechanics (gap -> final gap after adhesion)
4. Bonding Void/Open Risk (gap/overlay -> per-tile risk)
5. Anneal Delamination Risk (density gradient -> stress -> risk)
6. Yield Distribution (Monte Carlo aggregation of all risks)
7. Rules Compilation (threshold-based violation flags)

Errors compound through this chain. A 30% error in CMP recess can produce
a 2-5x error in void risk (depending on how many tiles are near the snap-in
threshold), which propagates to a 3x amplification in yield. The overall
chain amplification is the product of individual amplification factors.

### Fix Applied

1. **Error budget module** (`bondability/error_budget.py`): Computes
   per-solver uncertainty contributions and the aggregate chain
   amplification. Now uses data-driven analysis:
   - **Gap sensitivity fraction**: Computes the fraction of tiles within
     +/-2nm of the snap-in threshold (9nm). Tiles in this zone amplify CMP
     uncertainty into large risk changes. Tiles far from the threshold are
     insensitive to CMP errors.
   - **Correlation length mismatch**: Compares configured vs data-driven
     correlation length to estimate yield model uncertainty.

2. **Pipeline integration** (`bondability/pipeline.py`): The error budget is
   computed and saved as `error_budget.json` in every pipeline run. The
   log reports chain_amplification, confidence level, and dominant
   uncertainty source.

### Sensitivity of Final Yield to Each Solver

| Solver | Amplification | Dominant Source | Status |
|--------|-------------|-----------------|--------|
| Feature Extraction | 1.5x | Tile discretization | Calibrated |
| CMP Recess | 10x | Recess calibration curve | Literature |
| Contact Mechanics | 2-3x | Adhesion energy, range | Literature |
| Bonding Void/Open | 1-5x (data-driven) | Gap-to-risk logistic threshold | Literature |
| Anneal Delamination | 2x | Interface toughness Kc | Literature |
| Yield Distribution | 3x | correlation_length (UNCALIBRATED) | Uncalibrated |
| Rules Compilation | 1x | Threshold values | Literature |

**Critical path**: CMP -> Contact -> Bonding -> Yield. The chain
amplification through this path is typically 60-300x, meaning a 1% error
in CMP recess can become a 0.6-3% error in yield.

### Remaining Limitation

The error budget uses approximate heuristics, not rigorous propagation of
probability distributions through the solver chain. For production use,
consider implementing a full Bayesian UQ framework.

## Issue 4: Cu-Cu BONDING PHYSICS

### What Is Included

| Physics | Implementation | Notes |
|---------|---------------|-------|
| Surface roughness statistics | CMP recess sigma + MC perturbation | RMS roughness 0.3-0.8nm per trial |
| Post-CMP surface topography | Density-dependent bathtub CMP model | PCHIP interpolation, 3 presets |
| Contact mechanics | Dugdale cohesive zone + Kirchhoff plate | FFT-based spectral solver |
| Particle yield | Poisson model with critical area | Die-area-dependent |
| Overlay opens | erfc model with spatially-varying sigma | Edge/center sigma variation |
| Warpage | Beam mechanics with bonder correction | Residual intra-die quadratic |
| CTE mismatch stress | E_eff * delta_CTE * delta_T | Voigt/Reuss mixing rules |
| Edge stress | Shear lag model with exponential decay | Distance-from-edge weighting |
| Fracture mechanics | Griffith criterion via Hutchinson & Suo 1992 | Kc=0.7 MPa*sqrt(m), configurable flaw |
| Transient thermal shock | Lateral diffusion model | Confirms gradient is negligible |

### What Is Now Included (Added in This Fix)

| Physics | Implementation | Notes |
|---------|---------------|-------|
| Native oxide thickness | Exponential adhesion reduction | `oxide_thickness_nm` in BondingConfig |

The oxide model: `effective_adhesion = adhesion_energy * exp(-oxide_nm / 1.0)`.
At oxide=0nm: full adhesion. At oxide=2nm: adhesion reduced to ~14%, bonding
likely fails. This is a first-order model; real oxide effects depend on
plasma activation conditions and should be validated experimentally.

### What Is NOT Included (Known Gaps)

| Missing Physics | Impact | Mitigation |
|----------------|--------|------------|
| Grain growth kinetics | Cu grain growth during anneal (300C, 60min) converts van der Waals bonding to metallic bonding, increasing strength ~10x. Not modeled. | The current model conservatively uses initial adhesion energy. Post-anneal bond strength is higher than predicted. |
| Activation energy for Cu interdiffusion | Temperature-dependent Cu self-diffusion (Ea ~0.7-1.0 eV) controls grain growth rate. Not modeled. | Anneal time/temperature are config parameters but don't feed into a kinetic model. |
| Surface chemistry decomposition | Plasma activation, SAM treatment, and cleaning chemistry affect adhesion energy differently. Lumped into one `adhesion_energy` parameter. | For process optimization, decompose into per-treatment adhesion contributions. |
| Cu oxide chemistry | CuO vs Cu2O have different reduction kinetics during anneal. Model uses simple exponential; real behavior is more complex. | The `oxide_thickness_nm` parameter gives first-order sensitivity. |
| Non-planar pad topography | Real pads have dishing profiles (concave tops from CMP). The model uses tile-averaged gap. | Use smaller tiles (tile_um < 10) to better resolve intra-pad topography. |
| Bonding wave propagation | Bond front propagates from center outward; speed depends on local gap. Not modeled (steady-state contact only). | For most Cu-Cu designs at 4um pitch, steady-state is a good approximation. |

## Undisclosed Findings

### Issue 5: MC samples=200 Is Below Convergence Minimum

The default `monte_carlo_samples=200` throughout the codebase (config.py, process YAML files, validation benchmarks) is below the generally accepted minimum of 1,000 samples for Monte Carlo convergence of yield distributions. At 200 samples, the P10/P90 quantile estimates have high variance and are not statistically reliable. The yield_p50 (median) is more robust but still carries significant sampling noise. Users should increase `monte_carlo_samples` to at least 1,000 (preferably 5,000-10,000) for any quantitative yield prediction. The default of 200 was chosen for demo speed, not statistical rigor.

### Issue 6: p_void=1.0 for Low-Density Layouts

The logistic void risk model (`p_void = 1/(1 + exp(-k*(gap - snap_in)))`) produces p_void approaching 1.0 for all tiles when the gap significantly exceeds the snap-in threshold (9 nm). This is physically correct -- if the surface gap is much larger than the adhesion pull-in distance, bonding cannot occur. However, users should understand that p_void=1.0 is not a bug but rather the correct prediction for unrealistic layouts (e.g., 0.6% pad density producing ~15 nm gaps). The model transitions sharply around the snap-in threshold due to the logistic steepness parameter `k_void_logistic`. For layouts near the threshold, small changes in CMP recess or roughness can swing p_void dramatically, which is the root cause of the correlation_length sensitivity documented in Issue 2.

## Remaining Limitations (Summary)

1. **correlation_length_um is uncalibrated**: This single parameter dominates
   yield by ~100x. Without fab calibration, absolute yield values are
   unreliable. Use yield for relative comparison (layout A vs B) only.

2. **CMP calibration is from literature**: The `copper_hybrid_bonding` preset
   is from published data (Enquist 2019, Kim 2022), not from a specific fab.
   Users must replace with actual CMP measurements.

3. **No grain growth kinetics**: The model predicts initial bond quality but
   does not capture post-anneal bond strength improvement from Cu
   interdiffusion. This makes the model conservative (it underpredicts
   post-anneal bond quality).

4. **Chain error amplification**: The 7-solver chain amplifies input
   uncertainties by 60-300x through the critical path. The error budget
   module quantifies this but uses approximate heuristics.

5. **Tile resolution**: At 25 um tiles, the Dugdale cohesive zone (5 nm)
   is severely under-resolved. Use `refine_factor >= 4` in the contact
   solver for quantitative gap accuracy (16x slower).

6. **Archive data is stale**: The `archive/out_physics_final/` directory
   contains results from the old broken layout generator (0.6% density).
   These results are not representative of the current code.
